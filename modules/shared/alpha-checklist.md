# Station 18 — Alpha Readiness Checklisten

## Dashboard

### Routing

- [ ] Modul lädt über die zugewiesene Hash-Route
- [ ] Modul lädt erneut sauber bei wiederholtem Navigieren
- [ ] Alle Links innerhalb des Moduls funktionieren ohne 404

### UI Consistency

- [ ] Alle Überschriften nutzen die gemeinsamen `h1`/`h2` Klassen
- [ ] Abstände entsprechen den Vorgaben aus `shared.css`
- [ ] Nur gemeinsame Karten- und Button-Komponenten kommen zum Einsatz

### Empty-States

- [ ] Verwendet `ui-empty` Komponente
- [ ] Text lautet exakt „Keine Daten vorhanden.”
- [ ] Abstand zum umgebenden Inhalt bleibt konsistent

### Error-States

- [ ] Fehler erscheinen als `ui-notice`
- [ ] Konsolenmeldungen enthalten den passenden Fehlercode
- [ ] Keine unbehandelten Exceptions im Modul

### Detail Views

- [ ] Zeigt jede Entitäts-ID im Detailbereich
- [ ] Verwendet gemeinsame Karten für Inhalte
- [ ] Alle Links/Funktionen navigieren korrekt zu weiteren Ansichten

### List Views

- [ ] Listen nutzen das gemeinsame Listen-/Kartenlayout
- [ ] Empty-State verhält sich korrekt bei leeren Daten
- [ ] Klick auf einen Eintrag öffnet die zugehörige Detailansicht

### Form Views

- [ ] Nutzt gemeinsame Form-Row Bausteine
- [ ] Beschriftungen sind auf Deutsch und Pflichtfelder markiert
- [ ] Abbrechen- und Speichern-Buttons sind vorhanden und passend gestylt

### Console Cleanliness

- [ ] Keine unnötigen `console.log` Ausgaben (nur Fehlercodes)
- [ ] Keine Warnungen im Browser-Console Log
